# Documentation Style Guide

## General Rules
- Write in present tense ("Returns", not "Will return")
- First line is a summary sentence ending with period
- No blank line between summary and description
- Use backticks for parameter names, types, and code references
- Keep lines under 100 characters

## README Structure
1. Project name + one-line description
2. Prerequisites / dependencies
3. Quick start / installation
4. Usage examples
5. API reference (link to full docs)
6. Contributing guide
7. License

## What NOT to Document
- Private/internal functions (prefix `_`)
- Trivial getters/setters
- Self-explanatory code (redundant comments add noise)
